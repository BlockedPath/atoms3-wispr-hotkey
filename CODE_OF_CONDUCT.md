# Code of Conduct

## Our Standards

This project should be a practical, respectful place to improve firmware for a
small hardware device. Expected behavior includes:

- Be respectful and constructive in issues, reviews, and discussions.
- Focus criticism on code, hardware constraints, documentation, or behavior.
- Assume contributors have different levels of embedded, Bluetooth, and
  PlatformIO experience.
- Keep safety-sensitive hardware advice conservative and clear.

Unacceptable behavior includes harassment, personal attacks, discriminatory
language, publishing private information, or repeatedly derailing technical
discussion.

## Scope

This code of conduct applies in this repository, project issue trackers, pull
requests, and any official project discussion spaces.

## Enforcement

Project maintainers may remove comments, close issues, reject contributions, or
block participation when behavior is harmful to the project or its
contributors.

If you need to report a problem, contact the repository owner through GitHub.
Reports should include enough context for maintainers to understand what
happened.
